spacing rules

No spacing is allowed betweem te diamond brackets and the tag. Space in between value attributes is allowed and will be ignore by the browser.

Value Attributes

for example: [ p id="myId" ]
Value attributes have to be unique else compilation error.
Surround values with quotes as a string.
If using several quotes, remember to close the quotations properly.

Self-closing tag

For example: [p/]
Basically a shorthand notation for a tag that does not contain content.
But these are not allowed in HTML5 and [p][/p] needs to be used instead.

Structure of a html page

Head tag

Contains some metadata about the main content.
Can contain author's description of the page, page title, eternal resources needed to render the page etc.
For example set [meta charset = "utf-8"] tells the browser how to interpret the content of the webpage.
The title tag is required to be stated in the head tag.

Body Tag

Often referred to as a viewport, it is the root of all content visible to user.
Content goes inside the body.

HTML Content Models

Officially, HTML5 has 7 content types. 2 notable ones are Flow Content and Phrasing Content
Flow content == Block Element ; Phrasing Content == Inline elements

Block Elements

Allows other block elements and inline elements to exist within.
Requires a line on its own.

Inline Elements

Renders one the same line by default.

Semantic html Element

May help with search engine optimisation.

Heading Tags

Heading tags from 1 to 6
They signify the importance of the heading and convey the structure of the webpage.
They should not be used for styling purposes as the size can be overwritten by CSS.

Other semantic tags

1. Header Tag
Contains header information about the page.
The nav tag is contained within the header tag and contains links to different parts of the webpage.
2. Section and article tags
Usually, the section tag has article tags nested inside but it can work the other way round, depending on utility.
3. Aside tag
An element that contains information related to the main content but not a direct relationship.
4. Footer tag
Contains footer information.

All these tags are block level elements and they inherently contain information relating to the structure of the page.

Character Entity Reference

Characters such as <, > and & needs to be referenced as they are used as html tags.
The characters used to reference < is "& lt;" (without the space),
> is "& gt;",
and & is "& amp;"

Another common entity is the copyright symbol, ©. The reference is "& copy;".
Another one is the non-breaking space (nbsp), which groups the words together.
Quotation marks " are referenced by "& quot;"
